# MM6108 SoC データシート

Wi-Fi HaLow / IEEE 802.11ah · Sub-1 GHz 1/2/4/8 MHz 帯域幅 · MAC/PHY/Radio SoC

---

## 1 製品概要

### 1.1 はじめに

Wi-Fi HaLow (HEY-low と発音) は、IoT (モノのインターネット) のニーズに合わせて設計された初のグローバル Wi-Fi 規格 (IEEE 802.11ah) です。Sub-1 GHz の免許不要 RF 帯 (850–950 MHz) で動作し、従来の Wi-Fi よりも長距離、低消費電力、低コストを実現します。

Morse Micro は、世界をリードする Wi-Fi HaLow (802.11ah) ソリューションプロバイダーであり、複数の Wi-Fi HaLow 接続ソリューションを提供しています。MM6108 SoC は、IEEE 802.11ah 規格に準拠した無線、PHY、MAC 機能を統合し、最大 32.5 Mbps のデータレートをサポートするシングルチップソリューションです。無線部は 850 MHz から 950 MHz の間でプログラマブルに動作します。

MM6108 は、外部ホストへの Wi-Fi HaLow 接続をシンプルに実現するよう設計されています。RF インターフェースには、典型的な低消費電力・低コストデバイス向けにオンチップ増幅器を使用するオプションと、超長距離アプリケーション向けに PCB 実装の外付けパワーアンプを使用するオプションがあります。

### 1.2 特長

**無線**

- シングルストリーム最大データレート 32.5 Mbps (MCS=7、64-QAM、8 MHz チャネル、4 µs GI)
- 世界の Sub-1 GHz 周波数帯:850–950 MHz
- チャネル帯域幅オプション:1/2/4/8 MHz
- 最大出力電力:8 dBm

**802.11ah OFDM PHY**

- BPSK、QPSK、16-QAM、64-QAM 変調
- 自動周波数およびゲイン制御
- 前方誤り訂正 (FEC) の符号化・復号
- MCS レベル 0–7 および MCS 10
- 1 MHz 二重モード、Traveling Pilots、ショートガードインターバルをサポート

**802.11ah MAC (WFA HaLow 認証)**

- STA および AP の役割
- エネルギー検出付き Listen-Before-Talk (LBT)
- 802.11 省電力、フラグメンテーション、パケットアグリゲーション
- 長時間バッテリー寿命のための Target Wake Time (TWT)
- Restricted Access Window (RAW)
- 自動および手動 MCS レート選択

**ホストインターフェース**

- SDIO 2.0 スレーブ (DS 25 MHz / HS 50 MHz)、1 ビットおよび 4 ビットモード
- SPI モード動作をサポート

**電源管理**

- パワーダウン、ハイバネート、アクティブ Rx/Tx モード
- DC-DC コンバータ内蔵、3.0–3.6V 電源

**セキュリティ**

- AES 暗号化エンジン
- SHA-256、SHA-384、SHA-512 のハードウェアサポート
- 保護管理フレーム (PMF) 付き WPA3
- Opportunistic Wireless Encryption (OWE)

### 1.3 アプリケーション

監視カメラ、クラウド接続、低消費電力センサーネットワーク、ビルディングオートメーション、資産トラッキング、産業オートメーション、スマートホーム、EV 充電器、スマートシティネットワーク、Wi-Fi HaLow クライアントアダプター/ドングル、農業ネットワーク、スマートメーター、BT/ZigBee/Z-Wave から Wi-Fi HaLow へのゲートウェイなどの IoT および M2M アプリケーション。

## 3 機能の説明

### 3.1 機能ブロック図

MM6108 SoC は、Sub-GHz RF フロントエンド (LNA、PA、ADC、DAC)、RF PLL とデジタル PLL (32 MHz XTAL でクロック)、802.11ah PHY (FEC、OFDM、スクランブル/デスクランブル)、AES セキュリティ付きの 802.11ah MAC、Wi-Fi HaLow API レイヤ、SDIO/SPI スレーブインターフェース、電源管理およびディープスリープユニットを統合しています。

### 3.2 クロック

MM6108 は、XOP と XON の間に Pierce 発振回路で接続された 32 MHz の水晶を使用します。負荷容量 C1 と C2 は高精度 NP0 誘電体を使用してください:**C1 = C2 = 2 x CL**。

| パラメータ | Min | Max | 単位 |
| --- | --- | --- | --- |
| 周波数誤差 | −20 | \+20 | ppm |
| ESR | 10 | 50 | ohm |

### 3.3 電源管理

すべての電源は VBAT\_PWR および VBAT\_AON に供給される 3.0–3.6V 電源から取得されます。VDDIO は VBAT を超えてはなりません。内蔵バックコンバータは IND と VBUCK\_FB 間に 10 µH のインダクタ、および CAPR、CAPD、VBAT\_AON にセラミックデカップリングコンデンサが必要です。通常のブート動作のため、JTAG\_TRST は 10 kΩ 抵抗で GND にプルする必要があります。

#### 4.2.2 初期化

SPI モードに切り替えるには、電源投入後ホストが CS をローに保持したまま CMD0 を送信します。プロトコルの詳細は SD Specification Part E1 (「SDIO Simplified Specification」v2.00) を参照してください。

#### 4.2.3 バスタイミング

SPI クロックレートは最大 50 MHz をサポートします。タイミングは SDIO 規格と同じです。デフォルトは CPOL=0 で、初期化後に CPHA=0 に設定可能です。

### 4.3 GPIO

16 個の GPIO ピンはリセット時および省電力モード時にハイインピーダンスとなります。最小のスリープ電流を確保するため、未使用のピンはすべてプルアップまたはプルダウンしてください。

### 4.4 スリープ/ウェイクシーケンス

#### 4.4.1 ホストが MM6108 をスリープからウェイクさせる

1. ドライバが wake ピンをハイにし、10 ms 待ってから共有バスを初期化します。
2. 通信後、ドライバは 5–90 ms 待ってから wake ピンをロウにします。
3. wake ピンの立ち下がり後、MM6108 は動的な期間待機してからハードウェアスリープを開始します。

#### 4.4.2 MM6108 がホストをスリープからウェイクさせる (ホストインターフェース無効)

1. MM6108 がウェイクし、busy ピンをアサートします。
2. ホストは割り込みを受け、wake ピンをハイにし、10 ms 待ってホストインターフェースを有効にします。
3. MM6108 は busy をアサート後、すぐに通信を開始します。
4. 通信が完了すると busy が下がり、ホストは wake ピンを下げる前に待機します。
5. wake ピンの立ち下がり後、MM6108 はハードウェアスリープを開始します。

#### 4.4.3 MM6108 が通信を開始する (ホストインターフェース有効)

1. MM6108 は以前にホストによりウェイクされています。
2. MM6108 はホストにデータを返すために busy ピンをアサートします。
3. ホストは割り込みを処理しますが、それ以上の処理は不要 — wake ピンとホストインターフェースはすでにアクティブです。
4. 通信完了後、ハードウェアスリープを開始します。

## 11 取り扱いと保管

MM6108IQ は IPC/JEDEC J-STD-20 に従い **湿度感度レベル 3 (MSL3)** に分類されています。

防湿シール袋を開封した後、モジュールは以下のいずれかで取り扱う必要があります:

1. 30°C 以下、相対湿度 60% 未満の条件で **168 時間以内** に実装する、**または**
2. IPC/JEDEC J-STD-033 に従って継続的に保管する。

規格外の条件にさらされた IC は、実装前に IPC/JEDEC J-STD-033 に従って **ベーク** する必要があります。これに従わない場合、はんだリフロー中に修復不可能な損傷が発生します。
