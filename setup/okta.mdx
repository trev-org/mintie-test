The dedicated Okta connector is available from **Settings → Auth → SSO**. Pick **Okta** in the provider picker, paste your metadata URL or upload the metadata XML, and click **Connect**. See [Enable SSO](/setup/enable-sso) for the full walkthrough.

## What the connector offers

<CardGroup cols={2}>
  <Card title="Metadata URL or XML upload" icon="file-import">
    Paste your Okta app's metadata URL or upload the XML — no copy-pasting
    issuer, ACS, and signing certs.
  </Card>
  <Card title="Group claim mapping" icon="users-rectangle">
    Map Okta group claims to Mintie roles. Documented per Okta app type
    (OIDC web, SAML 2.0, custom).
  </Card>
  <Card title="CI compatibility" icon="circle-check">
    Mintie runs an automated E2E test against an Okta dev tenant on every
    release, so Okta-specific regressions don't reach you.
  </Card>
  <Card title="SCIM-ready" icon="arrows-rotate">
    Pairs with [SCIM 2.0 provisioning](/provisioning/scim) for
    auto-provision and deprovision from Okta.
  </Card>
</CardGroup>

## Connect Okta

<Steps>
  <Step title="Create the Okta SAML app">
    In Okta, create a new SAML 2.0 app for Mintie.
  </Step>
  <Step title="Copy your metadata">
    On the Sign-On tab, copy the metadata URL or download the metadata XML.
  </Step>
  <Step title="Paste it into Mintie">
    In Mintie, go to **Settings → Auth → SSO**, choose **Okta**, pick
    **Metadata URL** or **Metadata XML**, paste the value, and click **Connect**.
  </Step>
  <Step title="Run a test sign-in">
    Click **Test sign-in** to confirm the round-trip works before you turn on
    [Require SSO](/setup/enable-sso#require-sso-for-your-org).
  </Step>
</Steps>

### Fall back to generic SAML / OIDC

If you can't use the metadata flow, the [generic SAML / OIDC connector](/setup/generic-saml) accepts the same Okta values:

| Mintie field | Where to find it in Okta |
| --- | --- |
| Issuer | Okta app → Sign On → Issuer |
| Authorize URL | Okta app → Sign On → Authorize URL |
| Token URL | Okta app → Sign On → Token URL |
| JWKS URL | Okta app → Sign On → JWKS URL |
| Client ID / Secret | Okta app → General |
