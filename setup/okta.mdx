The dedicated Okta provider adapter is generally available. Connect your Okta tenant from the admin UI with a metadata URL, map group claims to Mintie roles, and rely on CI-tested compatibility on every release.

## What the Okta connector offers

<CardGroup cols={2}>
  <Card title="Metadata URL or XML upload" icon="file-import">
    Paste your Okta app's metadata URL or upload the XML — no copy-pasting
    issuer, ACS, and signing certs.
  </Card>
  <Card title="Group claim mapping" icon="users-rectangle">
    Map Okta group claims to Mintie roles. Documented per Okta app type
    (OIDC web, SAML 2.0, custom).
  </Card>
  <Card title="CI compatibility" icon="circle-check">
    Mintie runs an automated E2E test against an Okta dev tenant on every
    release, so Okta-specific regressions don't reach you.
  </Card>
  <Card title="SCIM-ready" icon="arrows-rotate">
    Pairs with [SCIM 2.0 provisioning](/provisioning/scim) for
    auto-provision and deprovision from Okta.
  </Card>
</CardGroup>

## Connect Okta

<Steps>
  <Step title="Create the Okta application">
    In the Okta admin console, create an OIDC web app or SAML 2.0 app for Mintie. Mintie supports both — pick the one your org standardizes on.
  </Step>
  <Step title="Copy the metadata URL">
    From the Okta app's **Sign On** tab, copy the metadata URL (SAML) or the OIDC discovery document URL.
  </Step>
  <Step title="Add the connection in Mintie">
    In the Mintie admin UI, go to **Settings → SSO → Add provider → Okta** and paste the metadata URL. Mintie resolves issuer, ACS, JWKS, and signing certs automatically.
  </Step>
  <Step title="Map group claims">
    Map Okta group claims to Mintie roles. Unmapped groups fall back to the org's default role.
  </Step>
  <Step title="Test, then enforce">
    Run a test sign-in from the connection's **Test** button. Once it passes, flip **Require SSO** for the org.
  </Step>
</Steps>

## Field reference

If you prefer to enter values manually instead of using the metadata URL, here's where each field lives in Okta:

| Mintie field | Where to find it in Okta |
| --- | --- |
| Issuer | Okta app → Sign On → Issuer |
| Authorize URL | Okta app → Sign On → Authorize URL |
| Token URL | Okta app → Sign On → Token URL |
| JWKS URL | Okta app → Sign On → JWKS URL |
| Client ID / Secret | Okta app → General |

<Note>
  The Okta adapter runs through WorkOS under the hood. See [How SSO works](/concepts/how-sso-works) for the full request flow.
</Note>
