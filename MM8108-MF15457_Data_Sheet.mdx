# MM8108-MF15457 Module Data Sheet

Multi-region Wi-Fi HaLow / IEEE 802.11ah Module · USB Host Interface and Integrated Amplifiers · Sub-1 GHz 1/2/4/8 MHz Bandwidth

---

## 1 Product Overview

### 1.1 Introduction

Wi-Fi HaLow (pronounced HEY-low) is the first global Wi-Fi standard (IEEE 802.11ah) specifically designed to meet the needs of the Internet of Things (IoT). It is an open-standard wireless technology operating in the sub-1 GHz license-exempt RF bands (850–950 MHz range), so it does not incur the ongoing monthly costs of cellular connections. Operating in the sub-1 GHz range lets this low-power protocol connect more IoT devices over much longer distances and with significantly lower power than traditional Wi-Fi.

The MM8108-MF15457 is a fully integrated Wi-Fi HaLow module with long range, low power consumption, and excellent RF performance, built around the second-generation MM8108 Wi-Fi HaLow SoC. It is compliant with IEEE 802.11ah, supports PHY rates up to 43.3 Mbps, and operates in the sub-1 GHz license-exempt RF bands worldwide, spanning 850 MHz to 950 MHz.

The MM8108's integrated power amplifier (PA) offers world-leading transmit performance and energy efficiency with on-chip amplification up to 26 dBm. An ultra-high-linearity integrated low-noise amplifier (LNA) provides exceptional receive sensitivity. Combined with digital transmit filtering and pre-distortion circuits, the result is an advanced transceiver that ensures out-of-the-box compatibility with all global Wi-Fi HaLow regulatory environments.

The module supports a USB 2.0 High-Speed host interface and an SDIO/SPI host interface, giving solution architects flexibility when integrating Wi-Fi HaLow into an existing design. With a footprint of only 11 mm x 10 mm, the MM8108-MF15457 showcases the cost and size benefits of the second-generation SoC, and it supports WPA3 and all standard security features required for Wi-Fi HaLow product certifications.

### 1.2 Features

**Radio**

- Single-stream maximum PHY rate of 43.3 Mbps at 8 MHz bandwidth
- Worldwide Sub-1 GHz frequency bands: 850–950 MHz
- Channel bandwidth options: 1/2/4/8 MHz
- High-power, high-efficiency integrated PA with maximum 26 dBm (400 mW) output power at 45% PA efficiency and 37% SoC efficiency
- Novel PA architecture maintains efficiency at power backoff
- Low-power receiver with integrated LNA, NF less than 4 dB

**802.11ah OFDM PHY (Wi-Fi Alliance Wi-Fi HaLow certified)**

- BPSK, QPSK, 16-QAM, 64-QAM and 256-QAM modulation
- Automatic frequency and gain control
- Packet detection and channel equalization
- Forward Error Correction (FEC) coding and decoding
- MCS levels 0–10
- 1 MHz duplicate mode and optional traveling pilots

**802.11ah MAC (Wi-Fi Alliance Wi-Fi HaLow certified)**

- Station (STA) and Access Point (AP) roles
- Listen-Before-Talk (LBT) access with energy detection
- 802.11 power save, fragmentation, and defragmentation
- Power-Saving Target Wake Time (TWT) for extended battery life
- Automatic and manual MCS rate selection
- On-chip software stack assisting host offload of Wi-Fi connection management

**Host Interface**

- USB 2.0 High-Speed compliant device interface (integrated PHY), up to 480 Mbps
- SDIO 2.0 High-Speed at 50 MHz max for 200 Mbps, 1-bit and 4-bit data modes
- SPI mode operation up to 80 MHz for 80 Mbps

**Power Management**

- Power Management Unit (PMU) with integrated DC-DCs and LDOs
- Single supply 3.0–3.6 V
- Multiple low-power modes (hibernate, deep sleep, snooze)

**Security**

- WPA3, Protected Management Frames (PMF), and Opportunistic Wireless Encryption (OWE)
- Hardware AES and SHA-2 (SHA-256, SHA-384, SHA-512) support

### 1.3 Applications

IoT and M2M applications including: Surveillance Cameras and Sensors, Cloud Connectivity, Low-power Sensor Networks, Building Automation, Asset Tracking, Building Access Control and Security, Drone Video and Navigation, Connected Toys, Rural Internet Access, Agricultural Networks, Utility Smart Meters and Intelligent Grid, Industrial Automation Controls, Smart Home Automation, EV Car Chargers, Appliances, Retail Point-of-Sale, Vehicle-to-Vehicle Communications, Biometric IDs and Keypads, Warehouse Connectivity, Intelligent Lighting Controls, BT/ZigBee/Z-Wave to Wi-Fi HaLow Gateways, Wi-Fi to Wi-Fi HaLow Bridges, Wi-Fi HaLow Client Adapters/Dongles, and Smart City Networks.

## 3 Functional Description

### 3.1 Block Diagram

The module integrates the MM8108 Wi-Fi HaLow SoC together with a 32 kHz crystal, a 32 MHz crystal, an antenna switch (SW), and the RF filters and matching components. The SoC exposes the host interfaces (SDIO, SPI, USB 2.0, UART) and is powered from VBAT, with VBAT\_TX feeding the transmit supply and VDD\_IO setting the digital I/O voltage. The antenna connects through pin ANT.

### 3.2 Power Supply Requirements

Module power is derived from a 3.0 to 3.6 V supply on pins VBAT and VBAT\_TX. VDDIO sets the I/O voltage of the module; it has an input range of 2.25 V to 3.6 V and should be connected to the same supply as the host MCU. There are no strict power-up sequencing requirements.

### 3.3 USB Host Interface

The module supports a USB 2.0 High-Speed host interface. The USB\_D\_P and USB\_D\_N lines connect to the host's USB data lines, with the module's supplies (VDD\_USB, VBAT, VBAT\_TX, VDDIO) decoupled with 10 µF and 100 nF capacitors. RESET\_N is driven from a host GPIO output (WIFI\_RESET\_N).

### 3.4 USB Network Adapter Interface

For a USB network adapter (dongle) design, VBUS from the USB-A connector feeds a 3.3 V regulator that supplies VDD\_3V3 to the module's VDD\_USB, VBAT, and VBAT\_TX rails. A power-on reset network (220 kΩ pull-up and 2.2 µF capacitor on WIFI\_RESET\_N) holds the module in reset until the supply is stable.

### 3.5 SDIO Host Interface

When interfacing via SDIO, ensure the host supports SDIO 2.0 with clock speeds up to 50 MHz; slower clocks reduce maximum throughput. The SDIO data and command lines should be pulled up with 10 kΩ to 100 kΩ resistors per the SDIO 2.0 specification.

For proper operation and to use the power-saving features, connect RESET\_N and WAKE to standard digital outputs (CMOS logic levels), and connect BUSY to a digital input (also CMOS levels). Do not use open-collector or open-drain circuits, as they will cause incorrect behavior.

In always-on applications where power-saving features are not used, such as access points, the WAKE pin can be left unconnected, reducing the host GPIO requirement to a single pin.

### 3.6 SPI Host Interface

When interfacing via SPI, the following are recommended for best throughput:

- The host must support level-triggered interrupts.
- The host must support full-duplex SPI mode.
- The host must support DMA-backed transactions on the SPI bus.

Standard SPI can achieve up to 25 Mbps at 50 MHz, which is significantly reduced without DMA support. For example, an SPI interface with an 8-byte buffer per transaction might only achieve 2 Mbps throughput on the bus.

As with SDIO, connect RESET\_N and WAKE to standard digital outputs and BUSY to a digital input (CMOS levels), and avoid open-collector or open-drain circuits. In always-on applications, the WAKE pin can be left unconnected.

## 9 Handling and Storage

The MM8108-MF15457 module is a moisture-sensitive device rated at **Moisture Sensitive Level 3 (MSL3)** per IPC/JEDEC J-STD-20.

After opening the moisture-sealed storage bag, modules subjected to reflow solder or other high-temperature processes must be:

1. Mounted to a circuit board within **168 hours** at factory conditions (30°C or less and less than 60% RH), **OR**
2. Stored continuously per IPC/JEDEC J-STD-033.

Modules exposed to conditions exceeding the packaging and storage limits **must be baked** before mounting, per IPC/JEDEC J-STD-033. Failure to meet packaging and storage conditions will result in irreparable damage during the solder reflow process.
