## Ce qui sera journalisé

Le tableau de bord présentera un flux filtrable d'événements d'identité pour l'organisation :

| Événement | Quand il est émis |
| --- | --- |
| `auth.signin.success` | L'utilisateur se connecte (mot de passe ou SSO) |
| `auth.signin.failure` | La connexion échoue, avec motif rédigé |
| `auth.sso.linked` | Un utilisateur Mintie a été lié à une identité IdP |
| `auth.sso.unlinked` | Lien supprimé (action administrateur) |
| `auth.session.forced_relogin` | Le job de migration de sessions a marqué une session |
| `scim.user.provisioned` | `POST /Users` SCIM a réussi |
| `scim.user.deactivated` | `DELETE /Users/{id}` SCIM a réussi |
| `apikey.created` | Clé d'API émise |
| `apikey.revoked` | Clé d'API révoquée (manuellement ou par désactivation IdP) |

## Interface

Le tableau de bord vivra dans **Paramètres → Authentification → Journaux d'audit**, filtrable par :

- Utilisateur
- Plage de temps
- Type d'événement
- Résultat (succès / échec)

Les lignes se déplient pour afficher les métadonnées complètes de l'événement (IP, user agent, claims IdP, etc.).

## Export

L'export CSV et une destination webhook pour le forwarding SIEM sont tous deux dans la liste de souhaits pour ce jalon.
