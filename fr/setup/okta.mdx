## Ce que le connecteur dédié offrira

<CardGroup cols={2}>
  <Card title="URL de métadonnées ou téléversement XML" icon="file-import">
    Collez l'URL de métadonnées de votre app Okta ou téléversez le XML — pas de copier-coller
    d'émetteur, ACS et certificats de signature.
  </Card>
  <Card title="Mappage de claims de groupe" icon="users-rectangle">
    Mappez les claims de groupe Okta aux rôles Mintie. Documenté par type d'app Okta
    (OIDC web, SAML 2.0, personnalisée).
  </Card>
  <Card title="Compatibilité CI" icon="circle-check">
    Mintie exécute un test E2E automatisé contre un tenant de développement Okta à chaque
    release, pour que les régressions spécifiques à Okta ne vous parviennent pas.
  </Card>
  <Card title="Prêt pour SCIM" icon="arrows-rotate">
    Se combine avec le [provisioning SCIM 2.0](/fr/provisioning/scim) pour
    l'auto-provisioning et le déprovisioning depuis Okta.
  </Card>
</CardGroup>

## Configuration aujourd'hui (chemin manuel)

Jusqu'à la publication du connecteur dédié, suivez le flux [SAML / OIDC générique](/fr/setup/generic-saml) avec ces valeurs spécifiques à Okta :

| Champ Mintie | Où le trouver dans Okta |
| --- | --- |
| Émetteur | App Okta → Sign On → Issuer |
| URL d'autorisation | App Okta → Sign On → Authorize URL |
| URL de jeton | App Okta → Sign On → Token URL |
| URL JWKS | App Okta → Sign On → JWKS URL |
| ID client / Secret | App Okta → General |
