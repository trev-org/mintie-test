Le connecteur Okta dédié est disponible dans **Paramètres → Authentification → SSO**. Sélectionnez **Okta** dans le sélecteur de fournisseur, collez votre URL de métadonnées ou téléversez le XML de métadonnées et cliquez sur **Connecter**. Consultez [Activer le SSO](/fr/setup/enable-sso) pour la procédure complète.

## Ce que le connecteur offre

<CardGroup cols={2}>
  <Card title="URL de métadonnées ou téléversement XML" icon="file-import">

  <Card title="Mappage de claims de groupe" icon="users-rectangle">

  <Card title="Prêt pour SCIM" icon="arrows-rotate">

  </Card>

## Connecter Okta

<Steps>
  <Step title="Créez l'app SAML Okta">
    Dans Okta, créez une nouvelle app SAML 2.0 pour Mintie.
  </Step>
  <Step title="Copiez vos métadonnées">
    Sur l'onglet Sign-On, copiez l'URL de métadonnées ou téléchargez le XML.
  </Step>
  <Step title="Collez-le dans Mintie">
    Dans Mintie, accédez à **Paramètres → Authentification → SSO**,
    choisissez **Okta**, sélectionnez **URL de métadonnées** ou **XML de
    métadonnées**, collez la valeur et cliquez sur **Connecter**.
  </Step>
  <Step title="Exécutez une connexion de test">
    Cliquez sur **Test sign-in** pour confirmer que l'aller-retour fonctionne
    avant d'activer [Exiger le SSO](/fr/setup/enable-sso#exiger-le-sso-pour-votre-organisation).
  </Step>
</Steps>

### Alternative : SAML / OIDC générique

Si vous ne pouvez pas utiliser le flux de métadonnées, le [connecteur SAML / OIDC générique](/fr/setup/generic-saml) accepte les mêmes valeurs Okta :

| Champ Mintie | Où le trouver dans Okta |
| --- | --- |
| Émetteur | App Okta → Sign On → Issuer |
| URL d'autorisation | App Okta → Sign On → Authorize URL |
| URL de jeton | App Okta → Sign On → Token URL |
| URL JWKS | App Okta → Sign On → JWKS URL |
| ID client / Secret | App Okta → General |
