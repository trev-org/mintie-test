Le connecteur Google Workspace dédié est disponible dans **Paramètres → Authentification → SSO**. Sélectionnez **Google Workspace**, saisissez votre domaine principal et approuvez les scopes SSO et Directory Sync en tant que super-administrateur Google. Consultez [Activer le SSO](/fr/setup/enable-sso) pour la procédure complète.

## Ce que le connecteur offre

<CardGroup cols={2}>
  <Card title="Connexion administrateur OAuth" icon="google">
    Un super-administrateur dans votre Google Workspace clique sur "Connecter Google" et
    accorde à Mintie un accès à scopes. Pas de XML SAML, pas de secrets clients à coller.
  </Card>
  <Card title="Synchronisation de l'annuaire" icon="people-arrows">
    Les utilisateurs et groupes sont synchronisés de Google vers Mintie chaque nuit. Les nouvelles embauches apparaissent
    automatiquement ; les départs sont désactivés sous 24 heures.
  </Card>
  <Card title="Statut de la connexion" icon="signal">
    Visible dans l'interface d'administration : heure de la dernière synchro, nombre d'utilisateurs, statut d'octroi des scopes.
  </Card>
</CardGroup>

## Connecter Google Workspace

<Steps>
  <Step title="Connectez-vous comme super-administrateur Google">
    Vous devez être super-administrateur de votre Workspace pour accorder les
    scopes SSO et Directory Sync.
  </Step>
  <Step title="Saisissez votre domaine principal">
    Dans Mintie, accédez à **Paramètres → Authentification → SSO**, choisissez
    **Google Workspace** et saisissez votre domaine principal (par exemple,
    `acme.com`).
  </Step>
  <Step title="Approuvez les scopes">
    Cliquez sur **Connecter**. Google ouvre un écran de consentement —
    approuvez les scopes SSO et Directory Sync de Mintie.
  </Step>
  <Step title="Exécutez une connexion de test">
    Cliquez sur **Test sign-in** pour confirmer que l'aller-retour fonctionne
    avant d'activer [Exiger le SSO](/fr/setup/enable-sso#exiger-le-sso-pour-votre-organisation).
  </Step>
</Steps>

### Alternative : OIDC générique

Si vous ne pouvez pas utiliser le connecteur dédié, le [connecteur SAML / OIDC générique](/fr/setup/generic-saml) accepte les endpoints OIDC standard de Google :

| Champ Mintie | Valeur |
| --- | --- |
| Émetteur | `https://accounts.google.com` |
| URL d'autorisation | `https://accounts.google.com/o/oauth2/v2/auth` |
| URL de jeton | `https://oauth2.googleapis.com/token` |
| URL JWKS | `https://www.googleapis.com/oauth2/v3/certs` |
| ID client / Secret | Depuis Google Cloud Console → APIs et Services → Identifiants |

<Tip>
  Sans le connecteur dédié, Google Workspace fonctionne pour le SSO mais pas
  pour l'auto-provisioning : les nouvelles embauches doivent être invitées
  manuellement jusqu'à l'activation de SCIM.
</Tip>
