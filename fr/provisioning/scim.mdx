## Ce que SCIM vous apporte

Quand votre IdP envoie un événement de cycle de vie utilisateur (nouvelle embauche, changement de groupe, déprovisioning), Mintie le reflète en quelques secondes — sans clic d'administrateur.

## Endpoints

L'URL de base est `https://app.mintie.example/scim/v2`.

| Méthode | Chemin | Objet |
| --- | --- | --- |
| `GET` | `/Users` | Liste les utilisateurs |
| `POST` | `/Users` | Crée un utilisateur |
| `GET` | `/Users/{id}` | Récupère un utilisateur |
| `PATCH` | `/Users/{id}` | Met à jour un utilisateur (y compris l'activation) |
| `DELETE` | `/Users/{id}` | Suppression logique d'un utilisateur |
| `GET` | `/Groups` | Liste les groupes |
| `POST` | `/Groups` | Crée un groupe |
| `PATCH` | `/Groups/{id}` | Met à jour l'appartenance à un groupe |
| `DELETE` | `/Groups/{id}` | Suppression logique d'un groupe |

## Authentification

Un jeton bearer par organisation. Générez et faites tourner le jeton depuis l'[interface d'administration](/fr/setup/enable-sso) une fois publiée.

```http
Authorization: Bearer scim_live_<token>
```

<Note>
  Les jetons SCIM sont limités à une organisation et à un seul usage. Ils ne peuvent ni lire
  ni écrire en dehors des endpoints SCIM.
</Note>

## Sémantique de la suppression logique

`DELETE /Users/{id}` ne supprime **pas** la ligne. Il définit `deactivated_at = now()` et révoque toutes les sessions. La ligne est conservée à des fins d'audit — voir [journaux d'audit](/fr/security/audit-logs).

## Conformité

L'endpoint SCIM cible la compatibilité avec la suite de conformité SCIM d'Okta. Une fois disponible, le SCIM de Mintie sera testable avec :

- Le testeur SCIM Okta
- L'agent de provisioning Microsoft Entra ID
- WorkOS Directory Sync (utilisé en interne)

## Ce que vous ferez

<Steps>
  <Step title="Générez un jeton SCIM">
    Depuis **Paramètres → Authentification → SSO → SCIM**, cliquez sur **Générer un jeton**.
  </Step>
  <Step title="Configurez votre IdP">
    Collez l'URL de base et le jeton dans les paramètres de provisioning SCIM de votre IdP.
  </Step>
  <Step title="Choisissez les mappages d'attributs">
    Les valeurs par défaut correspondent au SCIM Core standard. Mappez tous les attributs personnalisés dont
    votre organisation a besoin.
  </Step>
  <Step title="Envoyez un utilisateur de test">
    Provisionnez un utilisateur depuis l'IdP et confirmez qu'il apparaît dans Mintie.
  </Step>
</Steps>
