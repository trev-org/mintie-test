The SSO rollout is sequenced into four milestones. We're currently between M1 and M2.

## Milestones

<Steps>
  <Step title="M1 — OIDC happy path">
    **Done.** Schema migration and sign-in entry point.
    OIDC callback handler is being finished now (in development).
  </Step>
  <Step title="M2 — Okta + Google adapters GA">
    **In flight.** Native connectors for the two highest-priority IdPs.
    Generic SAML/OIDC works manually in the meantime.
  </Step>
  <Step title="M3 — SCIM + admin UI">
    **Next.** SCIM 2.0 endpoint and the admin settings page that lets
    customers self-serve SSO setup.
  </Step>
  <Step title="M4 — Password sunset">
    **Final.** Once an org has been on SSO for 60 days, password login is
    disabled. End-user comms ramp at T-30, T-7, and T-1 day.
  </Step>
</Steps>

## Driving constraints

The rollout is sequenced around three blocked enterprise deals:

- **Two on Okta** — M2's Okta connector is on the critical path
- **One needs SCIM** — M3's SCIM endpoint unblocks them
- **All three signed contracts** assuming GA by mid-July

## Out of scope

Two requests came up during scoping but were descoped — see the [FAQ](/migration/faq) for context:

- Direct LDAP / Active Directory integration
- Custom password complexity rules per org
