---
title: "SAML / OIDC genérico"
description: "Conecta cualquier proveedor de identidad SAML 2.0 u OIDC a Mintie"
---

Si tu IdP no es Okta o Google Workspace, el conector SAML / OIDC genérico lo cubre. Mintie enruta ambos protocolos a través de WorkOS, de modo que el código de la aplicación solo ve un callback con forma de OIDC.

<Note>
  La configuración hoy la hace tu equipo de cuenta de Mintie. Una vez se publique la
  [UI de administración](/es/setup/enable-sso), lo harás tú mismo.
</Note>

## Conexión OIDC

Necesitarás:

- Emisor (`iss`)
- URL de autorización
- URL de token
- URL de JWKS
- ID de cliente y secreto de cliente
- Scopes requeridos: Mintie necesita al menos `openid email profile`

## Conexión SAML 2.0

Necesitarás:

- XML de metadatos del proveedor de identidad (o la URL de metadatos)
- El certificado de firma del IdP (incrustado en los metadatos)
- Claims de atributos requeridos:
  - `email` (requerido)
  - `firstName` y `lastName` (opcional pero recomendado)
  - `groups` (opcional, usado para mapeo de roles)

## Endpoints de Mintie

Apuntarás tu IdP a:

| Endpoint de Mintie | URL |
| --- | --- |
| ACS (SAML) | `https://app.mintie.example/api/auth/saml/acs` |
| URI de redirección (OIDC) | `https://app.mintie.example/api/auth/sso/callback` |
| ID de entidad | `https://app.mintie.example/auth/<your-org-slug>` |

## Validación

Mintie valida la firma, el emisor, la audiencia y la expiración de cada callback. Los IdP mal configurados se reportan con códigos de error específicos, no un genérico "el inicio de sesión falló": consulta los [registros de auditoría](/es/security/audit-logs) cuando se publiquen.
